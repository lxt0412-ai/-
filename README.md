# AI 血糖健康管理助手 Demo

## 项目定位

这是一个 AI 产品经理作品集 Demo，用于展示 AI 在糖尿病/CGM 健康管理场景中的产品设计能力。

项目不接真实后端、不调用真实医疗接口、不接真实大模型 API，而是使用本地 JSON、本地模拟知识库、规则引擎和模板生成，模拟从患者数据采集到医生/健康管理师确认闭环的完整 AI 产品流程。

## 核心功能

- 患者建档
- 日常血糖/饮食/运动记录
- 血糖异常解释
- 风险分层
- 饮食个性化建议
- 运动个性化建议
- 高风险医生/健康管理师确认闭环
- 知识库与安全规则展示

## 工作流说明

患者档案读取 → 日常数据解析 → 单位标准化 → 知识库检索 → 风险分层 → 饮食建议生成 → 运动建议生成 → 安全审核 → 条件分流

条件分流规则：

- 低风险/中风险：生成患者端完整结果卡片
- 高风险/安全审核不通过：自动创建审核任务，进入医生/健康管理师审核台
- 专业端确认后：患者端展示“专业端确认版建议”

## 医疗安全设计

- 高风险不直接返回完整建议
- 药物调整类建议禁止输出
- 高风险任务进入专业端确认
- 输出保留免责声明
- 规则和知识库可追溯
- 安全审核拦截调药、停药、换药、明确诊断、保证没事、延误就医等表达
- 审核任务记录 case_id、风险原因、知识库证据、安全审核结果、审核状态和确认时间

## 技术栈

- React
- TypeScript
- Vite
- localStorage
- 本地规则引擎
- 本地模拟知识库

## 运行方式

```bash
npm install
npm run dev
```

启动后在浏览器打开 Vite 输出的本地地址，例如：

```text
http://127.0.0.1:5173/
```

## 演示场景

页面顶部提供 4 个一键演示按钮：

- 中风险餐后高血糖
- 高风险低血糖
- 轻度空腹异常
- CGM 快速下降

这些示例可用于面试中快速展示低中风险患者端卡片、高风险拦截、医生/健康管理师审核台和专业端确认版建议。
